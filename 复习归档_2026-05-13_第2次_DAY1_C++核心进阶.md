## 2. 速记版知识点

### 2.1 左值、右值与移动语义

最短判断：

- 左值：有稳定身份，通常能取地址，表达式结束后仍可访问。
- 右值：临时值或即将消亡的值，通常用于初始化或被移动。
- 左值引用：`T&`，通常只能绑定左值。
- 右值引用：`T&&`，通常绑定右值，用于移动语义和完美转发。

例子：

```cpp
int x = 1;
int& a = x;        // 左值引用
int&& b = 2;       // 右值引用绑定临时值
```

`std::move`：

- 本质是强制类型转换，把表达式转换成右值引用。
- 它本身不移动任何资源，真正移动发生在移动构造或移动赋值中。
- 被 `move` 后的对象仍然有效，但值处于“有效但未指定”状态。

高频陷阱：

```cpp
std::string s = "abc";
std::string t = std::move(s);
// s 仍可析构、可赋新值，但不应依赖 s 原来的内容。
```

移动构造 vs 拷贝构造：

- 拷贝构造通常复制资源，成本较高。
- 移动构造通常“偷走”资源所有权，成本较低。
- 对拥有堆内存、文件句柄、大 buffer 的类，移动语义很重要。

### 2.2 完美转发与 `std::forward`

模板中的 `T&&` 在类型推导场景下是转发引用，也叫万能引用。

```cpp
template <class T>
void wrapper(T&& value) {
    target(std::forward<T>(value));
}
```

重点：

- `std::forward<T>` 会保留实参原本的左值/右值属性。
- `std::move` 是无条件转成右值。
- `std::forward` 是有条件转发，只该用于转发引用场景。

引用折叠规则：

| 形式 | 折叠结果 |
|---|---|
| `T& &` | `T&` |
| `T& &&` | `T&` |
| `T&& &` | `T&` |
| `T&& &&` | `T&&` |

一句话：只要参与折叠的引用里有一个左值引用，结果就是左值引用。

### 2.3 RVO / NRVO

RVO：返回值优化，编译器直接在调用方存储位置构造返回对象，省掉临时对象。

NRVO：具名返回值优化，返回函数内具名局部对象时也可能省掉拷贝/移动。

```cpp
std::string make_name() {
    std::string name = "chip";
    return name; // 可能触发 NRVO
}
```

重点：

- C++17 起，某些返回临时对象场景的拷贝省略是强制的。
- `return std::move(local);` 反而可能阻止 NRVO，一般不要这样写。

## 3. 多态与虚函数

### 3.1 多态的核心条件

运行时多态通常需要：

1. 基类中有虚函数。
2. 派生类重写虚函数。
3. 通过基类指针或引用调用。

```cpp
struct Base {
    virtual void run();
};

struct Derived : Base {
    void run() override;
};

void call(Base& object) {
    object.run(); // 动态绑定
}
```

### 3.2 vptr / vtable

常见实现：

- 含虚函数的对象通常有一个隐藏的 vptr，指向虚函数表 vtable。
- vtable 中存放虚函数地址。
- 调用虚函数时，通过 vptr 找到实际类型对应的函数入口。

选择题关键词：

- “虚函数表是每个对象一份”通常错；一般是每个多态类一份，对象里保存 vptr。
- “虚函数调用一定静态绑定”错；通过基类指针/引用调用虚函数是动态绑定。
- “构造函数可以是虚函数”错；构造期间对象类型尚未完整形成。

### 3.3 虚析构函数

多态基类应提供虚析构函数。

```cpp
struct Base {
    virtual ~Base() = default;
};
```

原因：

- 如果基类析构不是虚函数，通过 `Base*` 删除 `Derived` 对象是未定义行为。
- 派生类资源可能无法正确释放。

### 3.4 纯虚函数与抽象类

```cpp
struct Device {
    virtual void submit() = 0;
    virtual ~Device() = default;
};
```

重点：

- 含纯虚函数的类是抽象类，不能直接实例化。
- 纯虚函数也可以有函数体。
- 派生类如果不实现所有纯虚函数，仍然是抽象类。

### 3.5 `override` / `final`

- `override`：要求该函数必须重写基类虚函数，签名不匹配会编译报错。
- `final`：禁止继续重写虚函数，或禁止类继续被继承。

建议：派生类重写虚函数时总是写 `override`。

### 3.6 虚继承与菱形继承

菱形继承问题：一个派生类通过多条路径继承同一个基类，可能产生多份基类子对象。

```cpp
struct A {};
struct B : virtual A {};
struct C : virtual A {};
struct D : B, C {};
```

虚继承作用：保证最底层派生类中共享同一份虚基类子对象。

代价：对象布局更复杂，访问虚基类成员可能有额外间接寻址。

## 4. 模板核心

### 4.1 函数模板与类模板

```cpp
template <class T>
T max_value(T a, T b) {
    return a < b ? b : a;
}

template <class T>
class Buffer {
public:
    T value;
};
```

重点：

- 模板不是普通函数/类本身，而是生成函数/类的蓝图。
- 模板实例化通常发生在编译期。
- 模板实现一般放在头文件中，否则容易链接失败。

### 4.2 可变参模板

```cpp
template <class... Args>
void log_all(Args&&... args) {
    (print(std::forward<Args>(args)), ...); // C++17 折叠表达式
}
```

重点：

- `Args...` 是模板参数包。
- `args...` 是函数参数包。
- 常用于转发构造参数、日志、tuple、容器 emplace。

### 4.3 SFINAE 与 `enable_if`

SFINAE：Substitution Failure Is Not An Error。模板替换失败不是硬错误，而是从候选集中移除该重载。

```cpp
template <class T, class = std::enable_if_t<std::is_integral_v<T>>>
void only_int(T value) {}
```

重点：

- 用于根据类型特征启用/禁用模板。
- C++20 concepts 让这类约束更直观。

### 4.4 Concepts

```cpp
template <class T>
concept Integral = std::is_integral_v<T>;

template <Integral T>
void only_int(T value) {}
```

优势：

- 约束更清晰。
- 错误信息更友好。
- 比复杂 SFINAE 更容易读。

## 5. STL 容器底层与迭代器失效

### 5.1 `vector`

特点：

- 连续内存。
- 支持 O(1) 随机访问。
- 尾插均摊 O(1)。
- 中间插入/删除 O(n)。

扩容：

- 容量不够时重新分配更大内存，再移动/拷贝旧元素。
- 扩容后，原来的指针、引用、迭代器全部失效。
- 常见扩容因子：GCC libstdc++ 常见 2 倍，MSVC 常见 1.5 倍；标准不规定。

### 5.2 `deque`

特点：

- 分段连续，不是整体一整块连续内存。
- 头尾插入删除通常 O(1)。
- 支持随机访问，但比 `vector` 多一次间接寻址。

常考：

- `deque` 适合双端队列。
- `vector` 更适合需要连续内存和 cache 友好的场景。

### 5.3 `map` vs `unordered_map`

| 容器 | 底层 | 有序性 | 平均复杂度 | 最坏复杂度 |
|---|---|---|---|---|
| `map` | 红黑树 | key 有序 | O(log n) | O(log n) |
| `unordered_map` | 哈希表 | 无序 | O(1) | O(n) |

重点：

- `map` 适合需要有序遍历、范围查询的场景。
- `unordered_map` 适合按 key 快速查找。
- 哈希冲突严重时，`unordered_map` 可能退化。

### 5.4 迭代器失效规则速记

`vector`：

- 扩容：所有迭代器、指针、引用失效。
- 未扩容的尾插：`end()` 失效，其他通常不失效。
- 中间插入/删除：插入/删除位置及其后的迭代器失效。

`list`：

- 插入不影响已有迭代器。
- 删除只使被删除元素的迭代器失效。

`map` / `set`：

- 插入通常不使已有迭代器失效。
- 删除只使被删除元素的迭代器失效。

`unordered_map`：

- 插入如果触发 rehash，所有迭代器失效。
- 删除只使被删除元素的迭代器失效。

## 6. C++11/14/17 高频特性

### 6.1 `auto`

- 根据初始化表达式推导类型。
- 默认会丢掉顶层 `const` 和引用。
- 想保留引用要写 `auto&` 或 `const auto&`。

```cpp
const int x = 1;
auto a = x;        // int
const auto b = x;  // const int
auto& c = x;       // const int&
```

### 6.2 lambda

```cpp
int factor = 2;
auto f = [factor](int x) { return x * factor; };
```

捕获方式：

- `[=]`：按值捕获。
- `[&]`：按引用捕获。
- `[this]`：捕获当前对象指针。
- C++14 支持泛型 lambda：`[](auto x) { return x; }`。

易错点：

- 按引用捕获局部变量时，lambda 逃逸出变量生命周期会产生悬空引用。
- 按值捕获默认不能修改捕获副本，除非 lambda 标记 `mutable`。

### 6.3 `constexpr`

- 表示可在编译期求值。
- C++11 限制较多，C++14 起函数体限制放宽。
- `constexpr` 函数在实参是常量表达式时可编译期求值，否则也可运行期执行。

### 6.4 结构化绑定

```cpp
std::pair<int, std::string> item{1, "one"};
auto [id, name] = item;
```

重点：

- C++17 引入。
- 可用于 pair、tuple、数组、聚合类型。
- 默认是拷贝；需要引用时写 `auto& [id, name] = item;`。

### 6.5 `optional` / `variant` / `any`

| 类型 | 用途 | 关键词 |
|---|---|---|
| `std::optional<T>` | 可能有值，也可能无值 | 替代“特殊值表示失败” |
| `std::variant<A, B>` | 类型安全的联合体 | 同一时刻持有候选类型之一 |
| `std::any` | 可持有任意可拷贝类型 | 类型擦除，运行期取回 |

常考：

- `optional` 访问前应判断是否有值。
- `variant` 通常用 `std::visit` 访问。
- `any_cast` 类型不匹配会抛 `std::bad_any_cast`，指针形式失败返回 `nullptr`。

## 7. 类型转换

### 7.1 `static_cast`

用于相对安全、编译期可检查的转换：

- 数值类型转换。
- 上行转换：派生类指针转基类指针。
- 明确的构造/转换操作。

不适合移除 `const`，不适合任意二进制重解释。

### 7.2 `dynamic_cast`

用于多态类型的安全下行转换。

```cpp
Base* base = get();
Derived* derived = dynamic_cast<Derived*>(base);
```

重点：

- 基类必须有虚函数，也就是多态类型。
- 指针转换失败返回 `nullptr`。
- 引用转换失败抛 `std::bad_cast`。
- 依赖 RTTI。

### 7.3 `const_cast`

用于添加或移除 `const` / `volatile`。

```cpp
const int x = 1;
int* p = const_cast<int*>(&x);
```

注意：

- 如果原对象本来就是 `const`，去 const 后修改是未定义行为。
- 如果原对象不是 const，只是通过 const 引用/指针传入，则去 const 修改原对象可以成立，但要谨慎。

### 7.4 `reinterpret_cast`

用于低层次重解释。

- 指针与整数之间转换。
- 不相关指针类型转换。
- 系统编程、驱动、协议解析中可能见到。

风险：

- 不保证类型安全。
- 容易违反对齐、别名规则或对象生命周期规则。
- 普通业务代码尽量避免。

## 8. 高频问答

### Q1：`std::move` 会移动对象吗？

答：不会。`std::move` 只是把表达式转换成右值引用，真正移动资源的是移动构造函数或移动赋值函数。

### Q2：为什么 `return std::move(local)` 可能不好？

答：局部对象直接 `return local` 时可能触发 NRVO；手写 `std::move(local)` 可能让编译器无法做 NRVO，只能走移动构造。

### Q3：为什么多态基类要有虚析构？

答：通过基类指针删除派生类对象时，需要动态绑定到派生类析构。如果基类析构不是虚函数，行为未定义，派生类资源可能泄漏。

### Q4：`vector` 扩容会发生什么？

答：重新分配更大连续内存，把旧元素移动或拷贝到新内存，再释放旧内存。扩容会导致原迭代器、指针、引用全部失效。

### Q5：`map` 和 `unordered_map` 怎么选？

答：需要有序遍历或范围查询选 `map`；只需要按 key 快速查找通常选 `unordered_map`。`unordered_map` 平均 O(1)，但哈希冲突严重时会退化。

### Q6：`dynamic_cast` 失败会怎样？

答：指针转换失败返回 `nullptr`；引用转换失败抛 `std::bad_cast`。它要求源类型是多态类型。

## 9. 选择题训练

### 题 1

关于 `std::move`，下列说法正确的是：

A. 它一定会释放原对象资源  
B. 它本质上是把表达式转换为右值引用  
C. 它会自动调用析构函数  
D. 它只能用于基本类型

答案：B

解析：`std::move` 本身不移动资源，只进行类型转换。

### 题 2

模板函数中用于保持实参左值/右值属性的工具是：

A. `std::move`  
B. `std::forward`  
C. `dynamic_cast`  
D. `sizeof`

答案：B

解析：完美转发依赖 `std::forward<T>` 保留值类别。

### 题 3

下列关于虚函数表的说法，正确的是：

A. 每个对象通常保存一整张虚函数表  
B. 含虚函数的对象通常含有 vptr  
C. 构造函数可以声明为虚函数  
D. 虚函数调用一定在编译期静态绑定

答案：B

解析：常见实现中，对象保存 vptr，vptr 指向类对应的 vtable。

### 题 4

通过基类指针删除派生类对象时，基类析构函数应该：

A. 是静态函数  
B. 是普通非虚函数  
C. 是虚函数  
D. 被删除 `= delete`

答案：C

解析：多态基类需要虚析构，保证派生类析构能被调用。

### 题 5

关于纯虚函数，下列说法错误的是：

A. 含纯虚函数的类是抽象类  
B. 抽象类不能直接实例化  
C. 纯虚函数一定不能有函数体  
D. 派生类未实现所有纯虚函数时仍是抽象类

答案：C

解析：纯虚函数可以在类外提供函数体。

### 题 6

`std::vector` 发生扩容后，原来的迭代器通常：

A. 全部仍然有效  
B. 全部失效  
C. 只有第一个有效  
D. 只在 debug 模式失效

答案：B

解析：扩容会搬迁底层连续内存，原迭代器、指针、引用全部失效。

### 题 7

`std::map` 的典型底层结构是：

A. 动态数组  
B. 哈希表  
C. 红黑树  
D. 单链表

答案：C

解析：`map` 通常基于红黑树，key 有序，操作 O(log n)。

### 题 8

`std::unordered_map` 在哈希冲突严重时，最坏查找复杂度可能退化为：

A. O(1)  
B. O(log n)  
C. O(n)  
D. O(n log n)

答案：C

解析：哈希表平均 O(1)，冲突严重时可能退化。

### 题 9

`dynamic_cast<Derived*>(base)` 转换失败时返回：

A. `nullptr`  
B. 随机地址  
C. 抛 `std::bad_alloc`  
D. 返回原指针

答案：A

解析：指针形式的 `dynamic_cast` 失败返回空指针。

### 题 10

移除对象 `const` 属性应该使用：

A. `static_cast`  
B. `dynamic_cast`  
C. `const_cast`  
D. `reinterpret_cast`

答案：C

解析：`const_cast` 专门用于添加或移除 cv 限定。

## 11. 易错结论速背

- `std::move` 不移动，只转右值。
- `std::forward` 保留值类别，主要用于完美转发。
- `return local` 可能触发 NRVO；不要习惯性写 `return std::move(local)`。
- 多态调用三要素：虚函数、重写、基类指针/引用。
- 多态基类析构函数要写 `virtual`。
- 构造函数不能是虚函数，析构函数可以是虚函数。
- `override` 是编译期检查神器，能防止签名写错。
- `vector` 扩容后原迭代器、指针、引用全部失效。
- `map` 红黑树、有序、O(log n)；`unordered_map` 哈希、无序、平均 O(1)。
- `dynamic_cast` 指针失败返回 `nullptr`，引用失败抛 `std::bad_cast`。
- `const_cast` 能去 const，但修改原本就是 const 的对象是未定义行为。
- `reinterpret_cast` 风险最高，偏底层系统编程。
