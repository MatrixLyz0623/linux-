## 2. 速记版知识点

### 2.1 程序内存布局

| 区域 | 存放内容 | 生命周期 | 常考点 |
|---|---|---|---|
| text/code | 机器指令、只读代码 | 程序运行期 | 通常只读，防止代码被修改 |
| rodata | 字符串常量、只读常量 | 程序运行期 | 字符串字面量通常在这里 |
| data | 已初始化的全局变量、静态变量 | 程序运行期 | `int g = 1;` |
| bss | 未初始化或初始化为 0 的全局/静态变量 | 程序运行期 | 可执行文件中不真实占同等大小 |
| heap | 动态分配内存 | 手动或智能指针管理 | `malloc/new`，容易泄漏/碎片化 |
| stack | 局部变量、函数调用栈帧 | 作用域结束自动释放 | 空间有限，递归过深会栈溢出 |

高频回答：

- 局部变量通常在栈上，`static` 局部变量在 data/bss。
- `malloc` 出来的内存在堆上，但指针变量本身可能在栈上。
- 字符串字面量如 `"abc"` 通常在只读区，不能通过 `char*` 修改。

### 2.2 `new/delete` vs `malloc/free`

| 对比项 | `new/delete` | `malloc/free` |
|---|---|---|
| 语言 | C++ 操作符 | C 标准库函数 |
| 构造/析构 | 会调用构造和析构 | 不会调用构造和析构 |
| 返回类型 | 返回具体类型指针 | 返回 `void*`，C++ 需强转 |
| 失败行为 | 默认抛 `std::bad_alloc` | 返回 `NULL` |
| 可重载 | 可以重载 `operator new/delete` | 不能按类重载 |

常考陷阱：

- `new[]` 必须配 `delete[]`，`new` 必须配 `delete`。
- `malloc` 分配 C++ 对象只拿到裸内存，不会构造对象。
- `free` 一个 `new` 出来的对象是未定义行为。

### 2.3 placement new

`placement new` 在一块已经分配好的内存上显式构造对象。

```cpp
#include <new>

alignas(T) unsigned char buffer[sizeof(T)];
T* object = new (buffer) T(args);
object->~T();
```

重点：

- 它不分配内存，只调用构造函数。
- 析构需要手动调用。
- 常用于内存池、容器底层、对象复用。

### 2.4 指针、引用和 const

`const` 读法：看 `const` 修饰的是谁。

```cpp
const int* p1;        // 指向 const int 的指针，不能通过 p1 改值，p1 可改指向
int const* p2;        // 同上
int* const p3 = &x;   // const 指针，p3 不能改指向，但可改 *p3
const int* const p4;  // 指针和指向的值都不能改
```

指针 vs 引用：

- 指针可以为空，引用理论上必须绑定有效对象。
- 指针可以改变指向，引用绑定后不能改绑。
- 指针有自己的地址和大小，引用通常是别名语义，底层实现可能仍用指针。
- 函数参数想表达“可选对象”用指针更自然；想表达“必须存在”用引用更自然。

野指针与悬空指针：

- 野指针：未初始化的指针。
- 悬空指针：指向对象已释放或生命周期已结束的内存。
- 解决：初始化为 `nullptr`，释放后置空，优先 RAII 和智能指针。

### 2.5 对象生命周期与 Rule of 0/3/5

对象生命周期包括：

1. 分配存储空间。
2. 构造对象。
3. 使用对象。
4. 析构对象。
5. 释放存储空间。

Rule of 3：如果类需要自定义析构、拷贝构造、拷贝赋值之一，通常三个都要考虑。

Rule of 5：C++11 后还要考虑移动构造、移动赋值。

Rule of 0：优先使用标准库资源管理类型，让类不手写析构/拷贝/移动。

面试回答模板：

> 只要类直接管理资源，比如裸指针、文件描述符、互斥锁句柄，就必须明确拷贝和移动语义。否则默认浅拷贝可能导致 double free、资源泄漏或悬空引用。工程里优先用 `std::unique_ptr`、`std::vector`、`std::string` 这类 RAII 类型，让编译器生成正确的特殊成员函数。

### 2.6 智能指针

`unique_ptr`：

- 独占所有权，不可拷贝，可移动。
- 适合表达“这个对象只有一个 owner”。
- 几乎没有额外开销，常作为默认选择。

`shared_ptr`：

- 共享所有权，控制块中维护引用计数。
- 拷贝会增加引用计数，析构会减少引用计数。
- 计数归零时释放对象。
- 控制块操作通常是原子的，但被管理对象本身不是线程安全的。

`weak_ptr`：

- 不拥有对象，不增加强引用计数。
- 用于打破 `shared_ptr` 循环引用。
- 使用前通过 `lock()` 尝试提升为 `shared_ptr`。

`make_shared` 优点：

- 一次内存分配同时放对象和控制块，性能更好，局部性更好。
- 异常安全性更好。

`make_shared` 注意点：

- 对象和控制块同一块内存，若还有 `weak_ptr` 存在，对象析构后整块内存可能延迟释放。
- 需要自定义 deleter 或私有构造场景时不一定适合。

## 3. 高频问答

### Q1：`malloc/free` 和 `new/delete` 最核心区别是什么？

答：`new/delete` 是 C++ 操作符，会调用构造函数和析构函数；`malloc/free` 只是申请和释放原始内存，不理解 C++ 对象生命周期。

### Q2：为什么析构函数通常要声明为虚函数？

答：当基类指针指向派生类对象，并通过基类指针 `delete` 时，如果基类析构不是虚函数，只会调用基类析构，派生类资源可能泄漏。这是多态基类的基本要求。

### Q3：`shared_ptr` 是线程安全的吗？

答：引用计数控制块的增减通常是线程安全的，不同 `shared_ptr` 实例在不同线程拷贝/析构一般没问题。但它管理的对象本身不是自动线程安全的，同时读写对象仍然需要加锁或其他同步。

### Q4：循环引用为什么会导致内存泄漏？

答：两个对象互相持有 `shared_ptr`，导致双方强引用计数都无法归零，即使外部引用都释放，对象也不会析构。解决方式是把其中一边改成 `weak_ptr`。

### Q5：为什么 C++ 里推荐 RAII？

答：RAII 把资源生命周期绑定到对象生命周期，构造时获取资源，析构时释放资源。这样能在异常、提前返回、多分支路径下自动释放资源，减少泄漏和重复释放。

## 4. 选择题训练

### 题 1

下列关于 C/C++ 程序内存布局的说法，正确的是：

A. 局部静态变量通常存放在栈区  
B. 未初始化的全局变量通常存放在 bss 区  
C. `malloc` 申请的内存通常存放在 data 区  
D. 字符串字面量一定存放在栈区

答案：B

解析：局部静态变量和全局变量一样具有静态存储期，通常在 data/bss；`malloc` 在堆；字符串字面量通常在只读数据区。

### 题 2

关于 `new/delete` 和 `malloc/free`，下列说法错误的是：

A. `new` 会调用构造函数  
B. `delete` 会调用析构函数  
C. `malloc` 只负责分配原始内存  
D. `free` 会自动调用 C++ 对象析构函数

答案：D

解析：`free` 不理解 C++ 对象生命周期，不会调用析构函数。

### 题 3

`const int* p` 的含义是：

A. `p` 是常量指针，不能改变指向  
B. `p` 指向的整数不能通过 `p` 修改，但 `p` 可以改变指向  
C. `p` 和 `*p` 都不能修改  
D. `p` 必须指向常量区

答案：B

解析：`const` 修饰的是 `int`，不是指针变量 `p` 本身。

### 题 4

`int* const p = &x;` 的含义是：

A. `p` 可以改变指向，但不能修改 `*p`  
B. `p` 不能改变指向，但可以修改 `*p`  
C. `p` 和 `*p` 都不能修改  
D. `p` 是指向常量整数的普通指针

答案：B

解析：`const` 在 `*` 右边，修饰指针变量本身。

### 题 5

下列哪种情况最可能导致悬空指针？

A. 指针初始化为 `nullptr`  
B. 指针指向 `new` 出来的对象，`delete` 后仍继续使用  
C. 使用引用作为函数参数  
D. 使用 `const int*` 指向整数

答案：B

解析：对象生命周期结束后，原指针仍保存旧地址，继续解引用就是悬空指针问题。

### 题 6

关于 `std::unique_ptr`，下列说法正确的是：

A. 可以拷贝，不可以移动  
B. 可以拷贝，也可以移动  
C. 不可以拷贝，可以移动  
D. 引用计数为 0 时释放对象

答案：C

解析：`unique_ptr` 表示独占所有权，不允许拷贝，但允许移动转移所有权。

### 题 7

关于 `std::shared_ptr`，下列说法正确的是：

A. 被管理对象本身自动线程安全  
B. 控制块引用计数通常是线程安全的  
C. `shared_ptr` 不需要控制块  
D. `shared_ptr` 一定比裸指针无额外开销

答案：B

解析：引用计数增减通常是原子的，但对象本身的并发读写仍需同步。

### 题 8

两个对象互相用 `shared_ptr` 持有对方，最可能出现的问题是：

A. 栈溢出  
B. 循环引用导致对象无法释放  
C. 编译必然失败  
D. 引用计数变成负数

答案：B

解析：互相持有强引用会使引用计数无法归零，需要用 `weak_ptr` 打破环。

### 题 9

`weak_ptr` 的主要作用是：

A. 独占对象所有权  
B. 增加强引用计数  
C. 观察对象但不拥有对象  
D. 替代所有裸指针运算

答案：C

解析：`weak_ptr` 不增加强引用计数，使用前通常通过 `lock()` 判断对象是否还存在。

### 题 10

关于 Rule of 0/3/5，下列说法正确的是：

A. 只要类中有普通 `int` 成员，就必须写析构函数  
B. 类直接管理资源时，应认真考虑拷贝、移动和析构语义  
C. Rule of 5 要求所有类都必须手写五个特殊成员函数  
D. 使用 `std::vector`、`std::string` 会破坏 RAII

答案：B

解析：直接管理资源时，默认拷贝可能导致浅拷贝、重复释放或资源泄漏。优先 Rule of 0。

## 6. 易错结论速背

- `malloc/free` 管内存，不管构造析构。
- `new/delete` 管对象生命周期，会调用构造析构。
- `new` 配 `delete`，`new[]` 配 `delete[]`，混用是未定义行为。
- `const int*`：值不能通过指针改，指针可换。
- `int* const`：指针不能换，值可改。
- 野指针偏“没初始化”，悬空指针偏“生命周期已结束”。
- `shared_ptr` 控制块线程安全，不代表对象线程安全。
- `weak_ptr` 不拥有对象，用来观察和打破循环引用。
- 直接管理资源才需要重点考虑 Rule of 3/5；最好用 RAII 类型做到 Rule of 0。
- `placement new` 只构造对象，不分配内存。
