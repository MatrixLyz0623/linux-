## 2. 虚拟内存基础

### 2.1 为什么需要虚拟内存

虚拟内存给每个进程提供独立、连续的虚拟地址空间。

好处：

- 隔离：进程之间不能随意访问彼此内存。
- 简化编程：每个进程都像拥有独立地址空间。
- 按需分配：虚拟地址不一定立刻对应物理内存。
- 支持 mmap、共享库、共享内存、COW。
- 支持换页和内存保护。

最短区分：

- 虚拟地址：程序看到的地址。
- 物理地址：真实内存硬件地址。
- 页表：虚拟地址到物理地址的映射表。
- MMU：CPU 中负责地址转换的硬件单元。

### 2.2 页 page

Linux 通常以页为基本内存管理单位。

常见页大小：

- 普通页：4KB 常见。
- 大页：2MB / 1GB 等，依赖架构和配置。

重点：

- 虚拟地址空间按页划分。
- 物理内存按页框 page frame 划分。
- 页表记录虚拟页到物理页框的映射和权限。

### 2.3 页表权限位

页表项常见信息：

- 物理页框地址。
- present：是否在内存中。
- read/write：是否可写。
- user/supervisor：用户态是否可访问。
- execute disable / NX：是否禁止执行。
- dirty：是否被写过。
- accessed：是否被访问过。

常考：

- 访问无权限页面会触发缺页异常或保护异常。
- 写只读页可能触发 COW。
- NX 位可防止数据页被当作代码执行。

## 3. 多级页表与 TLB

### 3.1 多级页表

x86-64 常见四级页表结构：

```text
PGD/PML4 -> PUD -> PMD -> PTE -> physical page
```

新平台也可能支持五级页表，扩大虚拟地址空间。

为什么多级：

- 虚拟地址空间很大，不可能为所有虚拟页一次性建立完整线性页表。
- 多级页表可以按需分配中间层，节省内存。

重点：

- 页表由内核维护，MMU 使用。
- 进程切换时通常会切换页表根。
- 内核空间映射通常在每个进程地址空间中有固定部分，但用户态不能访问内核页。

### 3.2 TLB

TLB：Translation Lookaside Buffer，地址转换缓存。

作用：缓存虚拟页到物理页的翻译结果，避免每次访存都走多级页表。

重点：

- TLB 命中：地址转换很快。
- TLB miss：需要 page table walk，成本更高。
- 进程切换可能导致 TLB 刷新或部分失效。
- 大页可以减少 TLB 压力，因为一个 TLB 项覆盖更大内存范围。

常考：TLB 缓存的是地址转换，不是普通数据缓存。

## 4. 缺页中断 page fault

### 4.1 page fault 是什么

CPU 访问某个虚拟地址时，如果页表项不存在、权限不符或需要特殊处理，会触发缺页异常，由内核处理。

常见原因：

- 第一次访问按需分配的匿名页。
- 访问 mmap 文件页，需要从文件加载。
- COW 写只读共享页。
- 访问非法地址。
- 权限错误，如写只读页、用户态访问内核页。

### 4.2 minor fault vs major fault

| 类型 | 含义 | 成本 |
|---|---|---|
| minor page fault | 不需要磁盘 I/O，例如页已在内存或 COW 分配 | 较低 |
| major page fault | 需要从磁盘读取页面 | 较高 |

重点：

- 缺页不一定是错误，有些是正常的按需分配。
- 非法访问导致的 page fault 可能转成 `SIGSEGV` 发给进程。
- 文件映射被截断后访问可能触发 `SIGBUS`。

## 5. 写时复制 COW

### 5.1 `fork` 为什么不立刻复制所有内存

`fork` 后父子进程看起来拥有独立地址空间，但内核不会立刻复制所有物理页。

COW 流程：

1. `fork` 后父子进程页表都指向同一批物理页。
2. 这些页被标记为只读/COW。
3. 父或子尝试写入时触发 page fault。
4. 内核复制该物理页。
5. 写入方页表指向新页，并恢复可写。

好处：

- `fork + exec` 很常见，很多时候子进程马上 `exec`，无需复制旧地址空间。
- 节省内存和时间。

### 5.2 COW 高频结论

- COW 不是父子进程共享变量修改。
- 父子进程写变量后互不影响。
- COW 发生在写入时，而不是 `fork` 时。
- COW 依赖页表权限和 page fault 实现。

## 6. 进程/线程内核视角

### 6.1 `task_struct`

Linux 内核用 `task_struct` 描述一个调度实体。

其中包含：

- 进程/线程状态。
- pid/tgid。
- 调度信息。
- 内存描述符 `mm_struct`。
- 打开文件表。
- 信号处理信息。
- 内核栈等。

重点：

- Linux 中线程和进程都用 `task_struct` 表示。
- 线程可以看作共享某些资源的 task。
- 同一进程内线程通常共享 `mm_struct`。

### 6.2 pid / tgid

| 名称 | 含义 |
|---|---|
| pid | 内核调度实体 ID，线程也有自己的 pid |
| tgid | 线程组 ID，通常等于主线程 pid，也就是用户看到的进程 ID |

常考：Linux 内核视角里线程也是 task；用户态常说的进程 ID 对应线程组 ID。

### 6.3 `clone()`

`fork`、`vfork`、pthread 创建线程底层都和 `clone()` 思路相关。

`clone()` 通过 flags 决定子 task 与父 task 共享哪些资源。

常见 flags：

| flag | 含义 |
|---|---|
| `CLONE_VM` | 共享地址空间 |
| `CLONE_FILES` | 共享文件描述符表 |
| `CLONE_FS` | 共享文件系统信息，如 cwd |
| `CLONE_SIGHAND` | 共享信号处理器 |
| `CLONE_THREAD` | 放入同一线程组 |

重点：

- 线程创建会共享地址空间等资源。
- 进程创建通常不共享地址空间，而是复制页表并配合 COW。

## 7. 调度器、nice 与 priority

### 7.1 CFS

CFS：Completely Fair Scheduler，Linux 普通进程默认调度器。

核心思路：

- 尽量让 runnable 任务公平获得 CPU 时间。
- 使用虚拟运行时间 vruntime 衡量任务已经获得的 CPU 时间。
- vruntime 小的任务更倾向于被调度。
- nice 值影响权重，权重影响 vruntime 增长速度。

重点：

- CFS 不是简单时间片轮转。
- nice 越小，优先级越高，获得 CPU 权重越大。
- 普通用户通常只能调低自己进程优先级，也就是增大 nice 值。

### 7.2 nice / priority

| 概念 | 说明 |
|---|---|
| nice | 用户可见的友好值，范围常见 -20 到 19 |
| priority | 内核调度优先级概念，和调度类相关 |
| real-time priority | 实时调度策略下的优先级 |

记忆：

- nice 值越低，越“不 nice”，越想抢 CPU。
- nice = -20 优先级高，nice = 19 优先级低。

## 8. 用户态 vs 内核态

### 8.1 为什么分用户态和内核态

CPU 有权限级别。普通应用运行在用户态，内核运行在内核态。

目的：

- 防止应用随意访问硬件和内核内存。
- 防止单个程序破坏整个系统。
- 由内核统一管理资源。

用户态不能直接做的事：

- 直接访问任意物理内存。
- 直接操作大多数硬件寄存器。
- 修改页表。
- 执行特权指令。

### 8.2 syscall 进入流程

系统调用是用户态请求内核服务的接口。

简化流程：

1. 用户程序把系统调用号和参数放到约定寄存器。
2. 执行 `syscall` 指令，CPU 切换到内核态。
3. 内核根据 syscall number 分发到对应处理函数。
4. 内核检查参数、权限，执行操作。
5. 返回用户态，把返回值放到寄存器。

历史/平台相关入口：

- `int 0x80`：老的软中断方式。
- `sysenter/sysexit`：较新的快速系统调用机制。
- `syscall/sysret`：x86-64 常见系统调用指令。

重点：

- 系统调用比普通函数调用成本高，因为涉及特权级切换和内核路径。
- 进入内核态不等于切换进程。
- 用户态通过 syscall 使用内核服务，如 `read/write/open/mmap/ioctl`。

## 9. 高频问答

### Q1：虚拟内存解决什么问题？

答：提供进程隔离和独立地址空间，支持按需分配、权限保护、mmap、共享库和 COW。

### Q2：TLB 是什么？

答：TLB 是地址转换缓存，缓存虚拟页到物理页的映射，减少多级页表查询成本。

### Q3：缺页中断一定是程序错误吗？

答：不是。按需分配、文件映射加载、COW 都会触发正常缺页；非法地址或权限错误才可能导致 `SIGSEGV`。

### Q4：COW 什么时候真正复制物理页？

答：`fork` 后父子先共享只读物理页，某一方写入时触发 page fault，内核才复制对应页。

### Q5：Linux 中线程和进程在内核里有什么共同点？

答：都由 `task_struct` 描述，线程是共享地址空间等资源的 task。

### Q6：`clone()` flags 的作用是什么？

答：决定新 task 和父 task 共享哪些资源，如地址空间、文件表、信号处理器、线程组等。

### Q7：nice 值越大优先级越高吗？

答：不是。nice 越小优先级越高；nice = -20 高，nice = 19 低。

### Q8：系统调用和普通函数调用有什么不同？

答：系统调用会从用户态切到内核态，由内核执行特权操作；普通函数调用仍在同一特权级内执行。

## 12. 易错结论速背

- 程序看到虚拟地址，硬件访问物理地址，MMU 用页表做转换。
- 页表记录映射和权限；TLB 缓存地址转换结果。
- TLB 不是数据缓存，TLB miss 是页表查询成本增加。
- 缺页不一定是错误，按需分配、mmap、COW 都可能正常缺页。
- COW 是 `fork` 后先共享，写入时复制。
- Linux 中进程和线程都用 `task_struct`，线程是共享资源的 task。
- `pid` 可指线程 ID，`tgid` 更接近用户看到的进程 ID。
- `clone` flags 决定共享什么资源，`CLONE_VM` 是共享地址空间。
- CFS 用 vruntime 追求公平，不是简单时间片轮转。
- nice 越小优先级越高，-20 高，19 低。
- 用户态不能执行特权指令或随意访问内核内存。
- syscall 从用户态切到内核态，但不等于一定切换进程。
