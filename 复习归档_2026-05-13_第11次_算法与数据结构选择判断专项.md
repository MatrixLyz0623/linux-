## 2. 复杂度速背

### 2.1 搜索与基础操作

| 算法/操作 | 平均复杂度 | 最坏复杂度 | 关键条件 |
|---|---:|---:|---|
| 顺序查找 | O(n) | O(n) | 无序/任意序列 |
| 二分查找 | O(log n) | O(log n) | 有序 + 随机访问 |
| 哈希表查找 | O(1) | O(n) | 平均 O(1)，冲突严重退化 |
| 红黑树查找 | O(log n) | O(log n) | 近似平衡 BST |
| AVL 查找 | O(log n) | O(log n) | 更严格平衡 |

易错：

- 二分查找的核心不是“数组”，而是有序和能高效访问中间元素。链表也能找中点，但不能保持 O(log n)。
- 哈希表不是“一定 O(1)”，最坏可 O(n)。

### 2.2 排序复杂度

| 排序 | 平均 | 最坏 | 空间 | 稳定 | 原地 |
|---|---:|---:|---:|---|---|
| 冒泡 | O(n^2) | O(n^2) | O(1) | 稳定 | 是 |
| 插入 | O(n^2) | O(n^2) | O(1) | 稳定 | 是 |
| 插入，已有序 | O(n) | - | O(1) | 稳定 | 是 |
| 选择 | O(n^2) | O(n^2) | O(1) | 不稳定 | 是 |
| 希尔 | 依赖 gap | O(n^2) | O(1) | 不稳定 | 是 |
| 归并 | O(n log n) | O(n log n) | O(n) | 稳定 | 否 |
| 快排 | O(n log n) | O(n^2) | O(log n)~O(n) | 不稳定 | 通常是 |
| 堆排 | O(n log n) | O(n log n) | O(1) | 不稳定 | 是 |
| 计数 | O(n+k) | O(n+k) | O(n+k) | 稳定 | 否 |
| 基数 | O(d(n+k)) | O(d(n+k)) | O(n+k) | 稳定 | 否 |

必背结论：

- 建堆 heapify 是 O(n)，不是 O(n log n)。
- 快排平均 O(n log n)，最坏 O(n^2)。
- 归并稳定但需要 O(n) 辅助空间。
- 堆排原地但不稳定。
- STL `std::sort` 是内省排序，不稳定；`std::stable_sort` 稳定。

### 2.3 图算法复杂度

| 算法 | 复杂度 | 用途 |
|---|---:|---|
| BFS | O(V+E) | 无权最短路、层序遍历 |
| DFS | O(V+E) | 连通性、拓扑、环检测 |
| Dijkstra + 堆 | O((V+E)logV) | 非负权单源最短路 |
| Bellman-Ford | O(VE) | 可处理负权边，能判负环 |
| Floyd | O(V^3) | 全源最短路 |
| Kruskal | O(ElogE) | 最小生成树 |
| Prim + 堆 | O(ElogV) | 最小生成树 |

易错：Dijkstra 不能处理负权边。

## 3. 排序稳定性与适用场景

### 3.1 稳定排序

稳定：相等关键字排序前后的相对顺序不变。

稳定排序：

- 冒泡排序
- 插入排序
- 归并排序
- 计数排序
- 基数排序

不稳定排序：

- 选择排序
- 希尔排序
- 快速排序
- 堆排序
- `std::sort`

### 3.2 场景选择

| 场景 | 常选算法 |
|---|---|
| 小数组或接近有序 | 插入排序 |
| 通用内存排序 | 快排/内省排序 |
| 要稳定 | 归并排序 / `stable_sort` |
| 链表排序 | 归并排序 |
| 外排序 | 归并排序 |
| Top K | 堆 / 快速选择 |
| 整数范围小 | 计数排序 |
| 定长整数/字符串 | 基数排序 |

选择题关键词：

- “关键字比较次数与初始序列无关”：选择排序、堆排序。
- “既适合顺序存储又适合链式存储”：插入排序、归并排序。
- “实际 STL sort”：内省排序，快排 + 堆排 + 插入排序混合。

## 4. 数据结构结论

### 4.1 线性结构

数组：

- 随机访问 O(1)。
- 中间插入删除 O(n)。
- cache 友好。

链表：

- 已知节点位置时插入删除 O(1)。
- 查找第 k 个节点 O(n)。
- 不支持高效随机访问。

栈：

- LIFO。
- 用于函数调用、括号匹配、表达式求值、DFS。

队列：

- FIFO。
- 用于 BFS、生产者消费者。

双端队列：

- 两端插入删除通常 O(1)。
- STL `deque` 分段连续，不是整体连续。

### 4.2 堆与优先队列

堆：

- `push/pop` O(log n)。
- `top` O(1)。
- 建堆 O(n)。

STL `priority_queue` 默认是大根堆。

Top K：

- 找前 K 大：维护大小为 K 的小根堆。
- 找前 K 小：维护大小为 K 的大根堆。

### 4.3 树

二叉树：

- n 个节点有 n+1 个空指针。
- 前序：根左右。
- 中序：左根右。
- 后序：左右根。
- 层序：BFS。

完全二叉树数组下标：

- 0-based：左孩子 `2i+1`，右孩子 `2i+2`，父节点 `(i-1)/2`。
- 1-based：左孩子 `2i`，右孩子 `2i+1`，父节点 `i/2`。

BST：

- 中序遍历得到升序序列。
- 平均查找 O(log n)，最坏退化 O(n)。

AVL：

- 任意节点左右子树高度差不超过 1。
- 查找严格 O(log n)。

红黑树：

- 近似平衡。
- 最长路径不超过最短路径 2 倍。
- 插入删除 O(log n)。

B+ 树：

- 数据库索引常用。
- 叶子节点有序链表，适合范围查询。

### 4.4 哈希与其他结构

哈希表：

- 平均 O(1)，最坏 O(n)。
- 冲突解决：链地址法、开放寻址。
- 负载因子过高会 rehash。

并查集：

- 路径压缩 + 按秩合并。
- 单次操作近似 O(1)。
- 用于连通性、Kruskal。

Trie：

- 用于前缀匹配。
- 时间复杂度和字符串长度相关。
- 空间开销较大。

跳表：

- 期望 O(log n)。
- Redis 有序集合曾使用跳表作为核心结构之一。

图存储：

- 邻接矩阵 O(V^2)，适合稠密图。
- 邻接表 O(V+E)，适合稀疏图。

## 5. 算法范式

### 5.1 分治

思想：分解问题，解决子问题，合并答案。

代表：

- 二分查找
- 归并排序
- 快速排序
- 大整数乘法
- 最近点对

### 5.2 动态规划 DP

适用特征：

- 最优子结构。
- 重叠子问题。

三要素：

- 状态定义。
- 状态转移方程。
- 边界条件。

代表：

- 背包
- LCS
- LIS
- 编辑距离
- 矩阵链乘

DP 与分治区别：DP 有重叠子问题，需要保存中间结果。

### 5.3 贪心

思想：每一步选择当前最优，希望得到全局最优。

代表：

- 哈夫曼编码
- Prim / Kruskal
- Dijkstra，非负权
- 活动选择

易错：贪心不一定总能得到最优解，需要证明贪心选择性质和最优子结构。

### 5.4 回溯与分支限界

回溯：

- N 皇后
- 子集
- 全排列
- 数独

分支限界：

- 0-1 背包
- TSP
- 常配合队列、优先队列和剪枝界限。

## 6. 数学、进制与位运算

### 6.1 组合数学

- 排列：`A(n,k) = n! / (n-k)!`
- 组合：`C(n,k) = n! / (k!(n-k)!)`
- 子集数：n 个元素有 `2^n` 个子集。
- 卡特兰数：n 个节点不同二叉树数量为 `C(2n,n)/(n+1)`。
- 抽屉原理：n+1 个物体放进 n 个抽屉，至少一个抽屉有两个。

### 6.2 补码

8 位补码范围：`[-128, 127]`。

重点：

- 正数原码、反码、补码相同。
- 负数补码 = 反码 + 1。
- 有符号整数溢出在 C/C++ 中是未定义行为。

### 6.3 位运算技巧

| 表达式 | 作用 |
|---|---|
| `x & (x - 1)` | 清除最低位的 1 |
| `x & -x` | 取最低位的 1 |
| `x ^ x` | 得 0 |
| `x ^ 0` | 得 x |
| `x & 1` | 判断奇偶 |
| `1 << k` | 2 的 k 次方，注意溢出 |

常见应用：

- 判断 2 的幂：`x > 0 && (x & (x - 1)) == 0`。
- 统计 1 的个数：循环清最低位 1。
- 找只出现一次的数：异或。

### 6.4 浮点数

IEEE 754 float：

- 1 位符号。
- 8 位阶码。
- 23 位尾数。
- 十进制有效精度约 7 位。

易错：浮点数比较不要直接用 `==` 判断数学意义上的相等。

## 7. C/C++ 算法题陷阱

### 7.1 数组退化

```c
int a[5] = {1, 2, 3, 4, 5};
int* p = a;

sizeof(a); // 20，假设 int 4 字节
sizeof(p); // 8，假设 64 位
```

函数参数中数组退化为指针：

```c
void f(int a[10]) {
    sizeof(a); // 8，假设 64 位
}
```

### 7.2 自增与优先级

- `i++` 返回旧值，之后自增。
- `++i` 先自增，返回新值。
- `*p++` 等价于 `*(p++)`。
- 短路求值：`&&` 左边为假右边不求值；`||` 左边为真右边不求值。

### 7.3 整数细节

- 整数除法向 0 截断。
- 负数取模符号跟被除数。
- `size_t` 是无符号类型，和 `int` 比较要小心。
- 左移有符号数溢出是未定义行为。

## 8. 高频问答

### Q1：快排最坏复杂度是多少？怎么避免？

答：最坏 O(n^2)，例如已有序且总取首/尾为 pivot。可用随机 pivot、三数取中或内省排序避免退化。

### Q2：建堆为什么是 O(n)？

答：虽然单次下沉是 O(log n)，但底层大量节点高度很低，从下往上累加是收敛级数，所以整体 O(n)。

### Q3：稳定排序有什么意义？

答：相同 key 的元素保持原相对顺序，适合多关键字排序，例如先按次关键字稳定排序，再按主关键字稳定排序。

### Q4：哈希表为什么最坏 O(n)？

答：如果大量 key 冲突到同一个桶，查找可能退化为遍历链表或探测序列。

### Q5：Dijkstra 为什么不能处理负权边？

答：Dijkstra 依赖“当前最短点一旦确定就不会再变”的贪心性质，负权边会破坏这个性质。

### Q6：B+ 树为什么适合数据库索引？

答：分叉多、树高低，减少磁盘 I/O；叶子节点链表适合范围查询。

### Q7：`x & (x - 1)` 有什么用？

答：清除最低位 1，可用于统计二进制 1 的个数和判断 2 的幂。

### Q8：函数参数里的数组 `sizeof` 为什么变成指针大小？

答：数组作为函数参数会退化为指针，因此 `sizeof` 得到的是指针大小。

## 9. 选择题训练

### 题 1

快速排序的最坏时间复杂度是：

A. O(n)  
B. O(n log n)  
C. O(n^2)  
D. O(log n)

答案：C

解析：pivot 选择极差时快排可能退化为 O(n^2)。

### 题 2

下列排序中稳定的是：

A. 快速排序  
B. 堆排序  
C. 归并排序  
D. 选择排序

答案：C

解析：归并排序稳定，快排/堆排/选择通常不稳定。

### 题 3

建堆 heapify 的时间复杂度是：

A. O(n)  
B. O(n log n)  
C. O(log n)  
D. O(n^2)

答案：A

解析：从最后一个非叶子节点向上调整，整体 O(n)。

### 题 4

STL `priority_queue<int>` 默认是：

A. 小根堆  
B. 大根堆  
C. 红黑树  
D. 队列

答案：B

解析：默认比较器 `std::less`，表现为大根堆。

### 题 5

BST 中序遍历得到：

A. 降序序列  
B. 随机序列  
C. 升序序列  
D. 层序序列

答案：C

解析：二叉搜索树中序遍历是升序。

### 题 6

Dijkstra 算法适用于：

A. 含负权边的任意图  
B. 非负权图的单源最短路  
C. 字符串匹配  
D. 排列组合计数

答案：B

解析：Dijkstra 不能处理负权边。

### 题 7

KMP 相比朴素匹配的主要优势是：

A. 主串指针不回溯  
B. 一定不用额外空间  
C. 只能匹配一个字符  
D. 时间复杂度 O(n^2)

答案：A

解析：KMP 通过 next 数组避免主串回退。

### 题 8

n 个元素的子集个数是：

A. n  
B. n^2  
C. 2^n  
D. n!

答案：C

解析：每个元素都有选/不选两种状态。

### 题 9

表达式 `x & (x - 1)` 的作用是：

A. 设置最高位 1  
B. 清除最低位 1  
C. 把 x 变成 0  
D. 判断浮点相等

答案：B

解析：这是位运算经典技巧。

### 题 10

函数参数 `void f(int a[10])` 内部 `sizeof(a)` 在 64 位系统上通常是：

A. 40  
B. 10  
C. 8  
D. 4

答案：C

解析：函数参数中的数组退化为指针。

## 11. 易错结论速背

- 快排平均 O(n log n)，最坏 O(n^2)。
- 归并稳定但需要 O(n) 额外空间。
- 堆排原地、不稳定；建堆 O(n)。
- `std::sort` 不稳定，`std::stable_sort` 稳定。
- 哈希表平均 O(1)，最坏 O(n)。
- 数组随机访问 O(1)，链表随机访问 O(n)。
- `priority_queue` 默认大根堆。
- BST 中序遍历升序，最坏退化 O(n)。
- 红黑树近似平衡，AVL 更严格。
- B+ 树叶子链表适合范围查询。
- Dijkstra 不能处理负权边。
- KMP 主串指针不回溯。
- DP 有重叠子问题，贪心需要证明正确性。
- `x & (x-1)` 清最低位 1，`x & -x` 取最低位 1。
- 函数参数数组退化为指针。
- 有符号整数溢出是 UB，浮点不要直接判等。
