## 2. 用户态内存分配器

### 2.1 `malloc` 不等于每次都系统调用

`malloc/free` 是 C 库分配器接口，通常先在用户态管理一大块堆内存。

重点：

- 小对象分配通常由分配器在用户态缓存和复用。
- 不一定每次 `malloc` 都调用 `brk` 或 `mmap`。
- 不一定每次 `free` 都立刻把内存还给操作系统。
- 频繁小对象分配可能受锁竞争、碎片、cache miss 影响。

### 2.2 `brk` vs `mmap`

| 方式 | 含义 | 常见用途 |
|---|---|---|
| `brk/sbrk` | 调整进程 heap 顶部 | 较小或连续堆扩展 |
| `mmap` | 映射一块独立虚拟内存区域 | 大块分配、独立映射 |

高频结论：

- heap 不是唯一动态内存来源，大块 `malloc` 可能来自 `mmap`。
- `free` 大块 mmap 内存时更可能直接 `munmap` 归还内核。

### 2.3 ptmalloc

glibc 默认分配器常称为 ptmalloc。

核心概念：

| 概念 | 含义 |
|---|---|
| arena | 分配区域，多线程下可有多个 arena 降低锁竞争 |
| bin | 按大小分类的空闲块链表 |
| fastbin | 小块快速回收链表，减少合并成本 |
| tcache | 线程本地缓存，glibc 新版本常见，减少多线程锁竞争 |
| top chunk | arena 顶部可扩展空闲块 |

重点：

- 多 arena 可以降低多线程 `malloc` 锁竞争，但也可能增加内存占用。
- tcache 是线程局部缓存，小对象分配释放更快。
- fastbin/tcache 都是小块优化，代价是碎片和安全复杂性。

### 2.4 jemalloc / tcmalloc

| 分配器 | 特点 |
|---|---|
| jemalloc | 多 arena、size class、碎片控制较好，常用于服务端 |
| tcmalloc | thread cache + central cache，强调多线程小对象分配性能 |

选择题记忆：它们都是用户态内存分配器，不是内核页分配器。

## 3. 进程地址空间布局

### 3.1 典型布局

进程虚拟地址空间常见区域：

```text
高地址
stack
mmap area: shared libraries, file mappings, anonymous mappings
heap
bss
data
text/code
低地址
```

常见区域：

| 区域 | 内容 |
|---|---|
| text | 代码段，只读可执行 |
| rodata | 字符串常量、只读数据 |
| data | 已初始化全局/静态变量 |
| bss | 未初始化或零初始化全局/静态变量 |
| heap | `malloc/new` 常用区域之一 |
| mmap area | 共享库、文件映射、大块匿名映射 |
| stack | 函数栈帧、局部变量、返回地址 |
| vdso/vvar | 内核映射给用户态的辅助页 |

重点：

- 栈通常向低地址增长，heap 通常向高地址增长，具体受架构和 ASLR 影响。
- ASLR 会随机化地址布局，提高攻击难度。
- `malloc` 的内存可能在 heap，也可能在 mmap 区。

### 3.2 `/proc/<pid>/maps`

`/proc/<pid>/maps` 展示进程虚拟内存区域。

示例字段：

```text
address           perms offset  dev   inode pathname
00400000-00452000 r-xp 00000000 08:01 12345 /bin/cat
```

字段含义：

| 字段 | 含义 |
|---|---|
| address | 虚拟地址范围 |
| perms | 权限：r/w/x + shared/private |
| offset | 文件映射偏移 |
| dev/inode | 对应文件设备和 inode |
| pathname | 文件路径或特殊区域，如 `[heap]`、`[stack]` |

权限位：

- `r`：可读。
- `w`：可写。
- `x`：可执行。
- `s`：shared。
- `p`：private，写时复制。

### 3.3 `vm_area_struct`

内核用 `vm_area_struct` 描述一段连续虚拟内存区域。

重点字段/概念：

- 起止虚拟地址。
- 权限 flags。
- 对应文件和 offset。
- 匿名映射或文件映射。

高频结论：一个进程地址空间不是一个连续“大数组”，而是一组 VMA 区域组成。

## 4. DMA 与一致性

### 4.1 DMA 是什么

DMA：Direct Memory Access。设备不经过 CPU 逐字节搬运，直接读写内存。

好处：

- 减少 CPU 参与。
- 提高大块数据传输效率。
- GPU/AI 加速卡、网卡、NVMe 都大量使用 DMA。

### 4.2 cache coherent vs non-coherent DMA

| 类型 | 含义 | 软件责任 |
|---|---|---|
| cache coherent DMA | CPU cache 与设备 DMA 访问保持一致 | 软件负担较轻 |
| non-coherent DMA | 设备 DMA 不自动和 CPU cache 一致 | 需要显式 cache flush/invalidate 或同步 API |

典型问题：

- CPU 写入 buffer 后数据还在 cache，设备 DMA 读内存可能读到旧值。
- 设备 DMA 写入内存后 CPU cache 中还有旧数据，CPU 读可能读到旧值。

解决思路：

- 使用 coherent DMA 内存。
- DMA 前后调用平台 DMA mapping API 做同步。
- 按方向区分 CPU-to-device、device-to-CPU、bidirectional。

### 4.3 `dma_alloc_coherent`

Linux 驱动中常见：

```c
void *cpu_addr = dma_alloc_coherent(dev, size, &dma_handle, GFP_KERNEL);
```

含义：

- 返回 CPU 可访问虚拟地址 `cpu_addr`。
- 返回设备可用 DMA 地址 `dma_handle`。
- 分配一致性 DMA 内存，CPU 和设备访问更容易保持一致。

重点：

- `cpu_addr` 给 CPU 用。
- `dma_handle` 给设备寄存器或 descriptor 用。
- 不能把普通 `malloc` 地址直接塞给设备当 DMA 地址。

## 5. IOMMU

IOMMU 类似“设备侧 MMU”，负责设备 DMA 地址到物理地址的转换和权限控制。

作用：

- 地址转换：设备看到 IOVA，IOMMU 转到物理地址。
- 隔离：限制设备只能访问授权内存。
- 支持虚拟化：不同虚拟机/进程隔离设备 DMA。
- 简化连续性：设备可看到连续 IOVA，背后物理页不一定连续。

高频结论：

- IOMMU 提升安全性，防止设备乱 DMA 破坏内存。
- 有 IOMMU 时，DMA 地址不一定等于物理地址。
- VFIO/DPDK/用户态驱动常关注 IOMMU 映射。

## 6. PCIe 基础

### 6.1 PCIe 拓扑

| 概念 | 含义 |
|---|---|
| Root Complex | CPU/内存系统连接 PCIe 总线的根 |
| Endpoint | PCIe 终端设备，如 GPU、AI 卡、网卡、NVMe |
| Switch | 扩展 PCIe 拓扑，连接多个设备 |
| Function | 一个 PCIe 设备中的功能单元 |

对于 AI 加速卡，通常是 PCIe Endpoint。

### 6.2 Config Space

PCIe 配置空间保存设备身份和资源信息。

常见内容：

- Vendor ID / Device ID。
- BAR 配置。
- capability 链表。
- MSI/MSI-X 能力。
- bus/device/function 编号。

驱动 probe 阶段会读取配置空间，识别设备并启用资源。

### 6.3 BAR 空间

BAR：Base Address Register。

作用：告诉系统设备需要映射的 MMIO 或 I/O 空间大小和类型。

驱动常见流程：

1. PCIe 枚举时系统为 BAR 分配地址。
2. 驱动 `pci_request_regions` 申请资源。
3. 驱动 `pci_iomap` 把 BAR 映射到内核虚拟地址。
4. 使用 `ioread32/iowrite32` 访问设备寄存器。

重点：

- BAR 常用于 MMIO 寄存器、doorbell、控制队列等。
- BAR 不是普通 RAM，访问要用 MMIO 语义。
- 用户态 runtime 可能通过驱动 `mmap` 映射部分 BAR 或 doorbell 页。

### 6.4 MSI / MSI-X

传统中断线共享复杂，PCIe 常用 MSI/MSI-X。

| 类型 | 特点 |
|---|---|
| MSI | Message Signaled Interrupt，设备通过写内存消息触发中断 |
| MSI-X | 支持更多中断向量，每个向量可独立配置 |

AI/GPU 驱动常用 MSI-X：

- 不同队列不同中断向量。
- 多核亲和性更好。
- 支持更细粒度事件通知。

### 6.5 TLP

TLP：Transaction Layer Packet，PCIe 事务层包。

常见类型：

- Memory Read。
- Memory Write。
- Completion。
- Configuration Read/Write。

高频记忆：PCIe 通信不是“直接函数调用”，而是通过事务包在链路上传输读写请求和完成响应。

## 7. 高频问答

### Q1：`malloc` 每次都会进入内核吗？

答：不会。分配器通常在用户态缓存和管理内存，只有需要扩展或归还大块内存时才可能调用 `brk/mmap/munmap`。

### Q2：tcache 的作用是什么？

答：tcache 是线程本地缓存，用于加速小对象分配释放，减少多线程锁竞争。

### Q3：`/proc/<pid>/maps` 能看什么？

答：能看进程虚拟内存区域、权限、映射文件、heap、stack、共享库、匿名映射等。

### Q4：共享内存为什么会出现在 maps 里？

答：共享内存本质上通过 mmap 映射到进程虚拟地址空间，所以会作为某个 VMA 出现。

### Q5：DMA 为什么需要关注 cache 一致性？

答：CPU 可能读写 cache，而设备直接读写内存；如果硬件不保证一致性，双方可能看到旧数据。

### Q6：IOMMU 有什么作用？

答：为设备 DMA 提供地址转换和访问隔离，防止设备访问未授权内存，也支持虚拟化和用户态驱动隔离。

### Q7：PCIe BAR 是什么？

答：BAR 是设备暴露 MMIO/I/O 资源的基地址寄存器，驱动通过映射 BAR 访问设备寄存器。

### Q8：MSI-X 相比 MSI 的优势是什么？

答：MSI-X 支持更多中断向量，且每个向量可独立配置，适合多队列设备。

## 8. 选择题训练

### 题 1

`malloc` 小对象分配通常：

A. 每次都必然系统调用  
B. 可能直接由用户态分配器缓存满足  
C. 一定直接操作物理内存  
D. 必然创建线程

答案：B

解析：分配器会缓存和复用内存，减少系统调用。

### 题 2

A. tcache  
B. TLB  
C. BAR  
D. TLP

答案：A

解析：tcache 是 thread cache，用于加速小块分配释放。

### 题 3

查看进程虚拟地址空间布局常用：

A. `/proc/<pid>/maps`  
B. `/etc/passwd`  
C. `/dev/null`  
D. `git log`

答案：A

解析：maps 展示 VMA 地址范围、权限、映射文件等。

### 题 4

内核中描述一段虚拟内存区域的结构是：

A. `vm_area_struct`  
B. `std::future`

答案：A

### 题 5

DMA 的含义是：

A. Direct Memory Access  
B. Dynamic Module Array  
C. Double Mutex Acquire  
D. Disk Main API

答案：A

解析：DMA 让设备直接读写内存。

### 题 6

non-coherent DMA 场景下，软件通常需要：

B. 做 cache flush/invalidate 或 DMA 同步  
C. 删除页表

## 10. 易错结论速背

- `malloc/free` 是用户态分配器接口，不等于每次系统调用。
- 小块常走 tcache/fastbin，大块可能走 `mmap`。
- tcache 减少多线程小对象分配锁竞争。
- `/proc/<pid>/maps` 看虚拟地址区域，不是直接看物理内存。
- `vm_area_struct` 描述一段连续虚拟内存区域。
- DMA 是设备直接读写内存，不是 CPU 搬运。
- non-coherent DMA 要显式 cache 同步。
- `dma_alloc_coherent` 的 CPU 地址和 DMA 地址用途不同。
- IOMMU 负责设备 DMA 地址转换和隔离，DMA 地址不一定是物理地址。
- PCIe Endpoint 是终端设备，Root Complex 连接 CPU/内存与 PCIe。
- BAR 暴露 MMIO/I/O 资源，驱动映射后访问寄存器。
- MSI-X 支持多中断向量，适合多队列设备。
